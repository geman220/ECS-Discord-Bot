{% call section_card("How each file is processed", icon="ti-route") %}
  • 1

    Read the title

    First # heading becomes the topic title. No heading → file is skipped with an error.

  • 2

    Resolve role access

    Names after **Role Access**: are matched to roles. Public assigns none (everyone sees it).

  • 3

    Create or update by title

    A matching title is overwritten; otherwise a new topic is created. The whole file body is stored as the topic content.

  • 4

    Commit & report

    Successful topics are saved together; per-file errors are reported back without blocking the rest of the batch.

File Format Requirements:

  • Files must have a .md extension
  • Each file must start with a heading (e.g., # Topic Title)
  • Include **Role Access**: Role Name to specify which roles can access the topic
  • Use **Role Access**: Public to make the topic visible to all users
  • Supported roles: Global Admin, Pub League Admin, ECS FC Coach, Pub League Coach, etc.
