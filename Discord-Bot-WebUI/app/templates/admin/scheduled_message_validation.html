How the System Works
  1. Message Creation: When matches are scheduled, messages are created with status='PENDING' and a scheduled_send_time
  2. Beat Schedule: Every 5 minutes, Celery Beat triggers the 'process_scheduled_messages' task
  3. Message Processing: The task finds all PENDING messages where scheduled_send_time <= now
  4. Queue for Sending: Each message is queued for sending and status changes to 'QUEUED'
  5. Discord Send: The send task posts to Discord and updates status to 'SENT' or 'FAILED'
  6. RSVP Collection: Users respond via Discord reactions, which update the database
