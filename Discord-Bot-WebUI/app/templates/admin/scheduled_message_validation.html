How the System Works
1. Message Creation

When matches are scheduled, messages are created with status='PENDING' and a scheduled send time

2. Beat Schedule

Every 5 minutes, Celery Beat automatically triggers the 'process_scheduled_messages' task

3. Message Processing

The task finds all PENDING messages where scheduled_send_time <= now and queues them

4. Discord Send

The send task posts messages to Discord and updates status to 'SENT' or 'FAILED'

5. RSVP Collection

Users respond via Discord reactions, which automatically update the database

6. Complete

Match day coordination is fully automated with real-time RSVP tracking
