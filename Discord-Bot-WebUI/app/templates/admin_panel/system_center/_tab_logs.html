These six sources are genuinely different things — read each for what it is.

  • App logs — the on-disk application log tailed from this container's filesystem (the only true tail; other workers/containers have their own file).
  • Audit trail — who-did-what admin actions, from the admin_audit_log database table.
  • API requests — /api/* traffic only, from the api_request_logs table (web page views and static assets are never logged).
  • Container — live docker logs straight from the Docker engine (Global Admin only).
  • Task errors — captured Celery tracebacks from failed background jobs (task_executions table).
  • Security — attack / ban events the detector wrote to the security_events table.

Heads up: the Clear old audit rows action (on the Audit trail source) deletes rows from the audit database table — it does NOT touch the on-disk application log files.
