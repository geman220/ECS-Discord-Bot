    {% call section_card('Flutter API Integration Reference', icon='ti-api') %}
    Share this with your Flutter developer

    All endpoints use base URL /api/v1. Most require JWT auth via Authorization: Bearer <token>.

    App Configuration (No Auth Required)
    Endpoint Method Description
    /app_config GET Returns min/latest build numbers, update URLs, force_update flag, and all feature toggles. Call before login to check for forced updates.
    Telemetry (Powers the Analytics Dashboard)

    Auth: JWT required. The analytics dashboard shows "no data" until the Flutter app sends these events.

    Endpoint Method Description Key Fields
    /telemetry/session POST Report session start/end. Send "start" on app open, "end" on background/close. session_id, event (start|end), platform, app_version, duration_seconds, screens_viewed
    /telemetry/screens POST Batch screen view events. Send when navigating between screens. Max 100 per request. session_id, screens[]: {screen_name, entered_at, exited_at, duration_seconds}
    /telemetry/feature-usage POST Track which features users interact with. Max 100 per request. events[]: {feature_name, used_at, platform, app_version}
    Error Reporting & Logging

    Auth: JWT + X-API-Key header.

    Endpoint Method Description Key Fields
    /analytics/errors POST Submit error/crash reports with context and recovery info. errors[]: {error_id, error_type, severity, error_message, device_info, app_version}
    /logs/mobile POST Submit structured logs (DEBUG/INFO/WARN/ERROR/FATAL). logs[]: {message, level, logger, trace_id, session_id, platform}
    Push Notifications

    Auth: JWT required.

    Endpoint Method Description
    /notifications/register POST Register FCM device token. Body: {device_token, platform: "ios"|"android", app_version}
    /notifications/unregister POST Deactivate device token on logout. Body: {device_token} (optional, deactivates all if omitted)
    /notifications/preferences GET PUT Get or update notification preferences (match_reminders, rsvp_reminders, team_updates, etc.)
    /notifications/test POST Send a test push to verify registration is working.
    Flutter Integration Checklist

    The admin dashboard will show real data once the Flutter app implements these:

    Call GET /api/v1/app_config on app startup (before login) to check for forced updates and fetch feature toggles
    Register FCM token via POST /notifications/register after login (include platform + app_version)
    Send POST /telemetry/session with event="start" on app open, event="end" on background/close
    Batch screen views via POST /telemetry/screens periodically or on session end
    Track feature usage via POST /telemetry/feature-usage when users interact with gated features
    Report errors via POST /analytics/errors (include X-API-Key header)
    Deactivate FCM token via POST /notifications/unregister on logout
