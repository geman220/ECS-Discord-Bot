    How this works

    Both the new membership system and the legacy system are running side by side, and every write goes to both, so they stay in sync. These switches only decide which one each subsystem reads from. On = new system (the default). Turn one Off to fail back to legacy instantly — no data is lost either way. Once the new system has run for a while with no issues, the legacy path can be retired.
