    How roles work here

    Creating a role: “New role” makes a Flask (app) role. You can also tick “also create a matching Discord role” to make both at once and link them. Need only a Discord role? Use “Create Discord role” on the Discord mapping tab.

    System roles (Global Admin, Pub League Admin, Discord Admin) are protected from deletion. A role that’s assigned to anyone can’t be deleted until it’s empty.

    Assigning to people happens in each person’s Member Hub — assign/remove here only opens that flow. Role changes queue a Discord sync automatically.

    What this matrix controls

    A permission gates fine-grained content (e.g. seeing a player’s admin notes, editing stats, reporting a match) — not whether someone can open an admin page (that’s role-based). Granting is per-role and applies to everyone with that role.

    Heads-up: a Redis-cached fast-auth path treats every permission as “admin-only,” so for a brief window after a change, cached sessions may not reflect a new grant until they refresh.
