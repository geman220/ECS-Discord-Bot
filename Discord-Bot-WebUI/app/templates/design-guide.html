Body Text

Default body text is 0.9375rem (15px) with a line height of 1.5. This provides good readability on both desktop and mobile devices.

Small text is 0.8125rem (13px) for secondary information.

Font Weights

Light (300): Used sparingly for large headlines

Regular (400): Used for body text

Medium (500): Used for emphasis and subheadings

Bold (700): Used for strong emphasis and primary headings

Usage Guidelines

  • Use primary buttons for primary actions (e.g., Submit, Save)
  • Use secondary or outline buttons for secondary actions (e.g., Cancel, Back)
  • Maintain consistent button ordering (e.g., primary action on the right)
  • Include icons only when they add meaningful context
  • Ensure buttons have adequate touch targets on mobile (min 44px height)

Usage Guidelines

  • Group related form fields logically
  • Use clear, concise labels that describe the expected input
  • Provide helpful error messages for validation failures
  • Ensure form controls are accessible (proper labeling, keyboard navigation)
  • Make sure forms are usable on mobile devices (adequate touch targets)
  • Consider using floating labels for a modern, clean look

Responsive Design

Our design system ensures a consistent experience across devices of all sizes.

Key Principles

  • Mobile-first approach: Design for mobile then enhance for larger screens
  • Flexible layouts: Use fluid grids that adapt to screen size
  • Touch-friendly interfaces: Ensure adequate touch targets (min 44px)
  • Responsive typography: Text remains readable across devices
  • Optimized images: Images scale appropriately for different screen sizes

Mobile Adaptations

  • Simplify navigation: Use hamburger menus, bottom navigation
  • Stack content: Convert multi-column layouts to single column
  • Optimize interactions: Larger touch targets, swipe gestures
  • Prioritize content: Show only essential information on smaller screens
  • Consider native app behaviors: Pull-to-refresh, install prompts
